# Startupilot 微信小程序

> 创业者服务平台 - 活动预约、学习资源、项目对接、VIP 会员体系（基于 uni-app + 微信云开发）

本仓库为 **Startupilot 创业者服务平台微信小程序** 的源码仓库，配套了一整套从「产品设计 → 技术架构 → 开发规范 → 部署运维 → 测试与数据导出」的 SOP 文档，适合：

- 想快速上手的前端 / 小程序开发者
- 需要评估可行性的产品经理 / 运营
- 负责部署、运维和数据导出的技术同学

> 如果你是第一次接触本项目，建议按本文档顺序阅读，并在需要深入时跳转到对应的专门文档或 SOP。


## 1. 项目概览

### 1.1 项目背景

Startupilot 是面向创业者的一站式服务平台，提供活动预约、学习资源、项目对接和会员体系等能力，通过微信小程序触达用户，降低使用门槛。

项目采用 **uni-app + 微信云开发** 的技术方案，兼顾开发效率与后期维护成本，适合中小团队快速迭代，当前目标版本为 **v1.0**。

### 1.2 目标用户与核心价值

**目标用户**
- 早期创业者 / 创业团队
- 投资人、产业合伙人
- 企业顾问、创业导师
- 对创业感兴趣的学习者

**核心价值**
- 聚合创业活动、学习资源与项目对接入口
- 提供 VIP 会员体系，提高用户黏性与付费转化
- 通过数据沉淀，逐步形成可复用的创业知识与资源网络

### 1.3 功能模块

**1）用户与会员系统**
- 微信一键登录，自动创建用户档案
- 用户基础信息维护（头像、昵称、UID）
- VIP 会员体系（普通会员 / NEO 会员）
- 兑换码激活、权益控制与统计

**2）活动预约（Activities）**
- 活动列表展示，支持线上/线下标签
- 活动详情：时间、地点、人数限制、说明等
- 活动报名 / 取消报名
- 「我的报名」查看个人预约记录

**3）学习资源（Learning）**
- 资源分类：案例分析 / 管理工具等
- 搜索过滤与 VIP 限制展示
- 收藏、查看详情与资源下载

**4）项目广场（Projects）**
- 发布项目：填写项目信息与需求
- 项目列表与筛选（按类型 / 标签 / 关键字过滤）
- 项目详情查看与「联系项目方」
- 「我的发布」管理个人发布记录

**5）个人中心（Profile）**
- 用户信息卡片与 VIP 卡片展示
- 数据统计：收藏数 / 发布数 / 报名数
- 常用入口：我的收藏、我的发布、我的报名、联系我们、公众号等

### 1.4 页面与导航结构

底部 TabBar 四个主入口：

- **首页（Home）**：活动推荐、关于 Startupilot、FAQ
- **学习（Learning）**：学习资源列表与搜索
- **项目（Projects）**：项目广场与发布入口
- **我的（Profile）**：账号信息、会员权益、个人数据与二级页面入口

二级页面包括：我的收藏 / 我的发布 / 我的报名 / 联系我们 / 项目发布页等。

### 1.5 项目状态与里程碑

- ✅ 需求与原型：完成主要业务流程的原型设计与交互稿
- ✅ 技术方案：确定 uni-app + 微信云开发的整体架构
- ✅ 核心开发：完成页面、组件、云函数与数据库设计
- 🚧 测试与优化：参考 `09_测试与质量保障.md` 执行测试与问题修复
- ⏳ 发布与运营：按 `08_部署与发布流程.md` 执行部署与上线


## 2. 技术架构与项目结构

本项目整体采用 **前端 uni-app + 微信云开发（Cloud Functions + Cloud Database + Cloud Storage）** 的 Serverless 架构。

### 2.1 整体架构

- **微信小程序客户端**
  - Pages（页面层）：首页 / 学习 / 项目 / 我的及其二级页面
  - Components（组件层）：卡片 / 按钮 / 表单 / 弹窗 / 加载 / 空状态等
  - Utils（工具层）：网络请求、数据处理、本地存储、日志上报等
- **微信云开发平台**
  - 云函数（Cloud Functions）：用户管理、活动管理、项目管理、学习资源、收藏、VIP 权限、管理员操作等
  - 云数据库（Cloud Database）：用户、活动、项目、资源、预约、收藏、兑换码等集合
  - 云存储（Cloud Storage）：图片、文档、资源文件等

详细的技术架构和数据流程请参考：`SOP文档/03_技术架构与数据流程.md` 和 `SOP文档/07_API 与数据模型说明.md`。

### 2.2 代码目录结构概览

> 以下为当前仓库核心目录的简化说明，详细开发规范见 `SOP文档/04_开发环境与目录结构.md` 和本仓库 `DEVELOPMENT.md`。

```bash
startupilot-miniapp/
├── pages/             # 页面目录（首页 / 学习 / 项目 / 我的 + 子页面）
├── components/        # 复用组件（玻璃卡片 / 按钮 / 列表卡片 / 弹窗等）
├── api/               # 前端接口封装与统一出口
├── utils/             # 工具方法（请求封装、存储、校验、日志等）
├── store/             # 状态管理（用户 / 活动 / 项目等）
├── static/            # 静态资源（图片、图标等）
├── styles/            # 全局样式（变量、混入、通用样式等）
├── cloudfunctions/    # 云函数（用户、活动、项目、学习、收藏、VIP 等）
├── App.vue            # 根组件
├── main.js            # 入口文件
├── manifest.json      # 小程序基础配置
├── pages.json         # 页面与路由配置
└── vite.config.js     # Vite 构建配置
```

### 2.3 主要技术栈

- **前端**：uni-app + Vue 3 + Composition API
- **状态管理**：Pinia
- **样式**：SCSS + 玻璃态（Glassmorphism）设计
- **构建工具**：Vite
- **后端与基础设施**：微信云开发（Cloud Functions + Cloud Database + Cloud Storage）

详细的字段设计、索引方案和集合定义，可参考：

- `SOP文档/06_云开发配置与数据库设计.md`
- `SOP文档/07_API 与数据模型说明.md`


## 3. 快速开始（本地开发）

这一节面向 **开发者**，帮你在 5–15 分钟内跑起来项目。更详细的开发说明见仓库内 `QUICK_START.md` 与 `DEVELOPMENT.md`。

### 3.1 环境准备

1. 安装 Node.js（建议 16+） 和 npm
2. 安装 HBuilderX / VS Code（推荐 VS Code + Volar）
3. 安装 微信开发者工具，并开启「云开发」能力
4. 克隆本仓库：

```bash
git clone https://github.com/ALEX152834/Startupilot-.git
cd startupilot-miniapp
npm install
```

### 3.2 基础配置

1. 在 `manifest.json` 中配置你自己的 **小程序 AppID**
2. 在 `App.vue` 或环境配置文件中替换为你自己的 **云开发环境 ID**
3. 按 `ENV_CONFIG.md` 配置必要的环境变量（如有）

### 3.3 启动开发环境

```bash
# 启动微信小程序端开发
npm run dev:mp-weixin
```

在微信开发者工具中：

1. 将项目目录指向 `dist/dev/mp-weixin`
2. 打开「云开发」，创建环境
3. 按 `CLOUD_STORAGE_QUICK_REFERENCE.md` 和 `ENV_CONFIG.md` 配置基础环境

### 3.4 初始化数据库与云函数

1. 按 `SOP文档/06_云开发配置与数据库设计.md` 与本仓库 `ENV_CONFIG.md` 创建数据库集合（例如：`users`、`events`、`projects`、`resources`、`bookings`、`favorites`、`vip_codes` 等）
2. 在云函数面板导入本仓库的 `cloudfunctions` 目录
3. 依次部署：用户、活动、项目、学习、收藏、VIP、管理员相关的云函数
4. 可以使用 `scripts` 目录下脚本或 `DATA_EXPORT_GUIDE.md` 中的说明导入初始测试数据

更详细的一步步操作可以参考：

- `QUICK_START.md` – 从零到可运行环境的完整步骤
- `DEPLOYMENT.md` – 部署到正式环境 / 预发布环境的操作指南
